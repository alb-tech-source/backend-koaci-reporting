# backend

Generated by `prisma-next init` with the Minimal template.

## First run

1. `npm run db:init` — applies your contract to the database (creates tables, signs the marker).
2. `npm run dev` — runs `src/index.ts`, which writes one row and reads it back.

## Available scripts

- `npm run dev` — run the Prisma Next sample script

### Database and migrations

- `npm run contract:emit` — emit contract artifacts after contract changes
- `npm run db:init` — initialize database state manually
- `npm run db:update` — update database state manually
- `npm run migration:plan` — create a migration plan
- `npm run migrate` — apply a planned migration
- `npm run db:seed` — insert sample users manually (run after `db:init`)

## Prisma Next

Prisma Next setup is scaffolded in:

- `src/prisma/contract.prisma`
- `prisma-next.config.ts`
- `prisma/db.ts`
- `src/lib/prisma.ts`

For provider-specific Prisma Next reference docs, see `prisma-next.md`. Prisma Next skills live in the upstream `skills/` directory: https://github.com/prisma/prisma-next/tree/main/skills.

Node-based Prisma Next projects expect Node.js 24 LTS or newer.
